WHAT'S NEW
KAM Sentinel
Here’s what changed since your last session.
  • 🎮 Live FPS Counter
    Real-time FPS via RTSS shared memory — plus 1% low and frametime (ms). Color-coded: green ≥60, yellow ≥30, red <30. No software to configure; works with any game RTSS supports.
  • 🔧 Forge Tab — Benchmarks & Fan Control
    Run CPU single-thread, multi-thread, RAM bandwidth, and disk benchmarks from the Forge tab. Fan presets (SILENT / BALANCED / PERFORMANCE / FULL_SEND) are persisted and applied via LibreHardwareMonitor.
  • 📁 Installer Fix — No More Permission Crashes
    When installed to Program Files, Windows blocks file writes — causing a silent crash on launch. User data (logs, profiles, backups) now lives in AppData where it belongs. Fresh installs are clean.
  • 🔊 Eve Voice Toggle
    The 🔊/🔇 button in the header lets you mute and unmute Eve’s voice at any time. Eve says goodbye before going silent and hello when she’s back. Wes Mode keeps her on always.
  • 👤 Wes Mode Identity Check
    On first launch, Eve checks whether you’re Wes Johnson. If yes, she skips the intro and delivers a personal message from Kris before starting your hearing calibration.
  • 🚨 Crash Recovery
    If KAM Sentinel crashes on startup, Eve logs the details and shows a friendly diagnosis the next time you launch — no raw error messages, no confusion. She’s already on it.
  • 🧵 Complete Crash Logging
    All unhandled exceptions — including crashes in background threads (GPU worker, hardware scheduler) — are now captured and written to crashes.jsonl. Previously, thread crashes vanished silently.
  • 🛡 Antivirus Startup Check
    On launch, Eve verifies that all required files are accessible before Flask starts. If an antivirus (Avast CyberCapture, Defender) is locking a file, she tells you exactly what’s blocked and how to fix it.
  • 📍 Dynamic Footer & Version Always Current
    The footer now shows the actual running version pulled from the server — no more stale hardcoded strings. Port number is also live. “Phase 1” quietly retired.
  • 🔄 Cache-Busting & Post-Update Reload
    Flask now sends no-store headers so Chrome never serves a stale dashboard. After an update installs, the page reloads automatically — no more “it updated but still shows old version”.
  • ✨ Smart What’s New
    This dialog now only shows updates you haven’t seen yet. Last-seen version is tracked in preferences.json so it survives reinstalls. Hit ✨ WHAT’S NEW in the header any time to review everything.
  • 🧹 Clean Build — Dev Scaffolding Removed
    Test Mode and hearing-calibration features have been extracted to their own project. The dashboard is now leaner — Eve’s persona and 404-alert popup remain fully intact.
